# User Personas

## Customer Personas

### P1: 신규 고객 (New Customer)
- **이름**: 김민수
- **역할**: 처음 방문한 잠재 고객
- **목표**: 원하는 상품을 빠르게 찾고, 신뢰할 수 있는 환경에서 첫 구매 완료
- **동기**: 합리적 가격, 간편한 구매 프로세스, 안전한 결제
- **불만 사항**: 복잡한 회원가입, 느린 페이지 로딩, 불투명한 결제 과정
- **기술 수준**: 중간 (모바일 쇼핑 익숙)
- **주요 접점**: Storefront, 회원가입, 상품 상세, 결제

### P2: 일반 고객 (Regular Customer)
- **이름**: 이서연
- **역할**: 반복 구매하는 기존 고객
- **목표**: 이전 주문을 쉽게 재주문하고, 주문/배송 상태를 실시간 확인
- **동기**: 편의성, 빠른 배송, 주문 이력 관리
- **불만 사항**: 재주문 번거로움, 배송 추적 불편, 반품 절차 복잡
- **기술 수준**: 중상 (웹/모바일 모두 사용)
- **주요 접점**: 셀프서비스 포털, 주문 내역, 배송 추적, 반품

### P3: VIP 고객 (VIP Customer)
- **이름**: 박지훈
- **역할**: 고빈도/고액 구매 고객
- **목표**: 개인화된 추천과 우선 서비스를 받으며 효율적으로 구매
- **동기**: 맞춤 추천, 빠른 처리, 특별 혜택
- **불만 사항**: 일반 고객과 동일한 대우, 관련 없는 추천, 재고 부족
- **기술 수준**: 상 (다양한 플랫폼 활용)
- **주요 접점**: AI 추천, 원클릭 재주문, 주소록, 실시간 알림

## Admin Personas

### P4: Operations Manager (운영 관리자)
- **이름**: 최영호
- **역할**: 주문 처리 및 배송 운영 총괄
- **목표**: 주문을 신속하게 처리하고 배송 SLA를 준수
- **동기**: 주문 처리 효율, 배송 비용 최적화, 고객 만족도
- **불만 사항**: 수동 주문 상태 관리, 배송 현황 파악 어려움, 실시간 정보 부족
- **기술 수준**: 중상
- **주요 접점**: 주문 관리, 배송 통합, 실시간 Dashboard, 주문 Lifecycle

### P5: Inventory Manager (재고 관리자)
- **이름**: 정수민
- **역할**: 재고 수준 관리 및 발주 의사결정
- **목표**: 적정 재고 유지, 품절 방지, 과잉 재고 최소화
- **동기**: 재고 최적화, 비용 절감, 수요 예측 정확도
- **불만 사항**: 실시간 재고 파악 불가, 수동 발주, 수요 예측 부재
- **기술 수준**: 중간
- **주요 접점**: 재고 Dashboard, AI 수요 예측, 자동 재주문 제안, AI Q&A

### P6: Finance Manager (재무 관리자)
- **이름**: 한지은
- **역할**: 매출/비용 분석 및 재무 보고
- **목표**: 정확한 매출 데이터와 재무 보고서 확보
- **동기**: 매출 추적, 비용 분석, ERP 연동
- **불만 사항**: 수동 데이터 집계, ERP 미연동, 실시간 매출 파악 불가
- **기술 수준**: 중간
- **주요 접점**: Analytics Dashboard, ERP 통합, 매출 보고서

## Warehouse Staff Persona (Phase 4)

### P7: Warehouse Staff (창고 직원)
- **이름**: 오태현
- **역할**: 입고/출고/재고 실사 수행
- **목표**: 모바일 기기로 빠르고 정확하게 재고 작업 수행
- **동기**: 작업 효율, 정확한 재고 기록, 오프라인 작업 지원
- **불만 사항**: PC 기반 작업 불편, 바코드 스캔 미지원, 오프라인 작업 불가
- **기술 수준**: 중하 (모바일 앱 사용 가능)
- **주요 접점**: 모바일 PWA, 바코드 스캐닝, 입고/출고 처리
